Trace — Introduction vulgarisée au fonctionnement de Godot

Une vision simplifiée de l’architecture de Godot afin de comprendre comment un moteur de jeu peut être construit.

Vision globale

Godot peut être vu comme une immense machine qui organise des objets, met à jour leur logique et les affiche à l’écran.

Le SceneTree

MainScene
├── Player
│   ├── Sprite
│   └── Collision
├── Enemy
├── Camera
└── UI

Toute la scène est un arbre. Chaque élément du jeu est un nœud relié à un parent.

Le système de Node

class Node
{
    List<Node> Children;
    Node Parent;

    virtual void Process(float delta)
    {
    }
}

Tous les objets du moteur héritent plus ou moins de cette idée.

La boucle de jeu

while (running)
{
    Input();
    Update();
    Physics();
    Render();
}

Le moteur répète cette boucle des dizaines de fois par seconde.

Le Renderer

Le renderer transforme les données du jeu en pixels visibles.

Les ressources

Les textures, shaders, scènes et sons sont chargés et partagés via un système de ressources.
